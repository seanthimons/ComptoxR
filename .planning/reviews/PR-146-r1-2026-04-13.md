## HIGH -- Must fix before merge

1. **All ToxValDB tests call `tox_*` but functions are exported as `toxval_*`** (`tests/testthat/test-tox_functions.R:6,44,57,65,74,99,115,122,128`, `test-tox_connection.R:76,92,96,111,132,315`) `[claude]`
   - The commit renamed functions from `tox_*` to `toxval_*` but the test files were not updated. Calls to `tox_results()`, `tox_sources()`, `tox_tables()`, `tox_fields()`, `tox_install()`, `tox_health()` will all fail with "could not find function".
   - Fix: Either rename the exported functions back to `tox_*` (matching the package convention `ct_*`, `eco_*`) or update every test call to use `toxval_*`. Given the commit message framing, renaming the tests to match `toxval_*` is likely the intent.

2. **Env var `"toxval_burl()"` read inconsistently without parens in two key locations** (`R/tox_connection.R:43`, `R/zzz.R:753`) `[claude]`
   - `zzz.R` sets the env var as `"toxval_burl()"` (with literal parentheses) on lines 498, 504, 516, 518, 519, 520, 527, 531. But `tox_connection.R:43` reads `Sys.getenv("toxval_burl")` (no parens) and `zzz.R:753` also reads `Sys.getenv("toxval_burl")`. These two reads will always return `""` because the correct key is `"toxval_burl()"`. The connection path detection on `tox_connection.R:44` (`grepl("\\.duckdb$", toxval_burl)`) will silently never match, falling back incorrectly.
   - Fix: Standardize on one name. Recommend dropping the parens — rename all setters/getters in `zzz.R` to use `"toxval_burl"` (without parens), which is the conventional format for R env var keys.

3. **`human_eco` filter crashes at runtime — column absent from v9.7.0 schema** (`R/tox_functions.R:378-380`) `[claude]`
   - `toxval_results()` accepts a `human_eco` parameter and executes `dplyr::filter(.data$human_eco %in% he_vals)` (line 380), but the test file (`test-tox_functions.R:108-116`) explicitly documents that `human_eco` does not exist in the v9.7.0 database. Any call to `toxval_results(human_eco = ...)` will abort with a column-not-found error from DuckDB.
   - Fix: Remove the `human_eco` parameter from the function signature, documentation, and internal plumbing (`tox_functions.R:267,295,321,399,408`) until the schema includes it. Re-add when upgrading to a schema version that has the column.

4. **GitHub Actions workflows missing `permissions: contents: write`** (`.github/workflows/db-dsstox.yml`, `db-ecotox.yml`, `db-toxval.yml` — all jobs) `[claude]`
   - All three workflows use `gh release upload` to attach assets to releases, but no `permissions` block is declared. GitHub Actions defaults to read-only for `contents` on many org/repo configurations. The upload step will fail with a 403 error.
   - Fix: Add to each job:
     ```yaml
     permissions:
       contents: write
     ```

5. **`data-raw/pt.R` uses `polite` and `tidyverse` packages not declared in DESCRIPTION** (`data-raw/pt.R:18-22`) `[claude]`
   - `library(tidyverse)` and `library(polite)` are called, but neither package appears in `DESCRIPTION` Imports or Suggests. Running `devtools::install(dependencies = TRUE)` will not install them; any CI run or user attempting to regenerate the periodic table data object will fail with "there is no package called 'polite'".
   - Fix: Add `polite` and `tidyverse` (or granular tidyverse components) to `Suggests` in DESCRIPTION.

## MEDIUM -- Should fix

6. **CI: Unvalidated empty TAG causes cryptic downstream failures** (`.github/workflows/db-dsstox.yml:39-41`, `db-ecotox.yml:39-41`, `db-toxval.yml:39-41`) `[claude]` — If no releases exist, `gh release list | jq '.[0].tagName'` returns `null`/empty; subsequent `gh release view ""` fails with an unhelpful error. Add `[ -z "$TAG" ] && echo "No releases found" && exit 1` after tag resolution.

7. **Plumber `/search` endpoint: unbounded `limit` allows DoS** (`inst/plumber/toxval/plumber.R:51`) `[claude]` — `limit` has no upper bound; a client can pass `limit = 2147483647`, forcing a full table scan and OOM crash. Add: `if (limit_int > 10000L) { res$status <- 400; return(list(error = "limit exceeds 10000")) }`.

8. **Plumber `/search` endpoint: unhandled R errors expose stack traces** (`inst/plumber/toxval/plumber.R:49-52`) `[claude]` — No `tryCatch` wraps the endpoint handler. Database errors surface as raw R tracebacks in the HTTP 500 response, leaking internal paths and query details. Wrap in `tryCatch(..., error = function(e) { res$status <- 500; list(error = conditionMessage(e)) })`.

9. **Plumber has no localhost-only middleware** (`inst/plumber/toxval/plumber.R`) `[claude]` — Documentation implies localhost-only use, but nothing enforces this at the network layer. If the Plumber process is exposed (e.g., via port forwarding or misconfigured deployment), all endpoints are unauthenticated and publicly readable. Add a `@filter localhost_only` plumber filter that checks `req$REMOTE_ADDR` and rejects non-loopback connections with 403.

10. **CI: Missing validation that Rscript build actually created the DB file** (`.github/workflows/db-dsstox.yml:~95`, `db-ecotox.yml:~95`, `db-toxval.yml:~95`) `[claude]` — After running the R build step, `DB_PATH` is computed via `Rscript -e "cat(...)"`. If Rscript exits non-zero or the build creates no file, `DB_PATH` is empty and `gh release upload` silently tries to upload a nonexistent path. Add: `[ -z "$DB_PATH" ] || [ ! -f "$DB_PATH" ] && echo "DB not found at $DB_PATH" && exit 1`.

11. **`data-raw/pt.R` does not validate API key before making authenticated requests** (`data-raw/pt.R`) `[claude]` — The script calls `dss_query()` and `ct_chemical_list_search_by_name()` which require `ctx_api_key`, but never checks `Sys.getenv("ctx_api_key")` at startup. Users get a cryptic 401/403 far into a multi-minute run. Add upfront: `if (!nzchar(Sys.getenv("ctx_api_key"))) stop("ctx_api_key not set.")`.

12. **`dss_install()`/`eco_install()` fallback silently returns dest even if build fails** (`R/dss_connection.R:145-157`, `R/eco_connection.R:128-140`) `[claude]` — The error handler calls `.dss_build_from_source(dest)` but the outer function then returns `invisible(dest)` unconditionally. If the build also fails, the error is swallowed and the caller receives a path to a file that doesn't exist. Move `invisible(dest)` inside the success path, or verify `file.exists(dest)` before returning.

13. **CI: Date parsing in staleness check has no error handling** (`.github/workflows/db-dsstox.yml:65`, `db-ecotox.yml:65`, `db-toxval.yml:65`) `[claude]` — `ASSET_EPOCH=$(date -d "$ASSET_DATE" +%s)` will silently produce an empty value if `$ASSET_DATE` is malformed or empty. The subsequent arithmetic `$(( (NOW_EPOCH - ASSET_EPOCH) / 86400 ))` then errors or produces nonsense, leading to wrong staleness decisions. Add `2>/dev/null` and validate `$ASSET_EPOCH` is non-empty before arithmetic.

## LOW -- Nice to have

Extract to a module-level constant and reference it from both diagnostic functions.

16. **Unused `.get_con()` closure in `plumber.R`** (`inst/plumber/toxval/plumber.R:16`) `[claude]` — The function is defined but never called; all endpoints reach the connection implicitly via `tox_*()` functions. Dead code adds confusion about connection lifecycle. Remove the definition and add a comment explaining that connections are managed by `ComptoxR:::.tox_get_con()`.

17. **Ambiguous DTXSID fallback in `pt.R` takes first match without logging candidates** (`data-raw/pt.R:343-350`) `[claude]` — When multiple DTXSIDs match a PubChem CID, the script warns and silently picks the first one. The warning does not log which DTXSIDs were candidates, making it impossible to audit the selection. Print the candidate list to the console when interactive.

18. **`--clobber` upload leaves no audit trail in CI** (`.github/workflows/db-dsstox.yml:~98`, `db-ecotox.yml:~98`, `db-toxval.yml:~98`) `[claude]` — There is no log line distinguishing "new asset created" from "existing asset replaced". Add a pre-upload check that logs which action is taken.

## Verified correct

- **Backward compatibility of `dss_install()`/`eco_install()` changes**: New `build` and `tag` parameters both default to values that preserve prior behavior. Existing call sites require no changes.
- **`.db_download_release()` error handling**: Handles 3-attempt retry, cleans up partial downloads on error, returns clear messages on 404.
- **`.onUnload()` includes `toxval` cleanup**: `zzz.R:831` calls `.tox_close_con()`, which correctly handles NULL connections. ToxValDB connections are released on package unload.
- **Post-build file existence check**: Both `dss_connection.R` and `eco_connection.R` verify `file.exists(dest)` after build completes and abort with a clear message if the file is missing.
- **VCR cassettes provided**: New test files include pre-recorded fixtures for AMOS endpoints, enabling offline/CI test runs without live API calls.
- **`toxval_path()` respects option override**: `getOption("ComptoxR.toxval_path")` is checked before defaulting to `tools::R_user_dir()`, consistent with the pattern used by `dss_path()` and `eco_path()`.
