# Requirements: ComptoxR Stub Generation Pipeline

**Defined:** 2026-02-12
**Core Value:** Generated API wrapper functions must send requests in the format the API expects — correct body encoding, content types, and parameter handling.

## v1.9 Requirements

Requirements for Schema Check Workflow Fix. Each maps to roadmap phases.

### CI Fix

- [x] **CI-01**: User can run `pkgload::load_all()` in CI without `usethis` dependency error
- [x] **CI-02**: Unicode map data is built via `data-raw/` script, not executed at source time in `R/`
- [x] **CI-03**: `R/unicode_map.R` is removed from `R/` directory (moved to `data-raw/`)
- [x] **CI-04**: Package `sysdata.rda` contains the unicode_map object after running `data-raw/` script

### Schema Diffing

- [x] **DIFF-01**: User can see which specific endpoints were added, removed, or modified (not just file-level changes)
- [x] **DIFF-02**: User can see whether each change is breaking (removed endpoint, changed params) or non-breaking (new endpoint)
- [x] **DIFF-03**: Auto-generated PR body includes structured endpoint-level diff summary with breaking/non-breaking classification

### Reliability

- [x] **REL-01**: Schema download functions have configurable timeout protection to prevent hung API calls from blocking the workflow
- [x] **REL-02**: Workflow exits cleanly with a warning (not failure) when schemas cannot be downloaded
- [x] **REL-03**: Workflow distinguishes expected 404s from brute-force path discovery vs actual API failures
