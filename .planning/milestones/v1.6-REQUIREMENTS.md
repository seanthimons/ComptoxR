# Requirements: ComptoxR v1.6 Unified Stub Generation Pipeline

**Defined:** 2026-01-29
**Core Value:** Generated API wrapper functions must send requests in the format the API expects

## v1.6 Requirements

Requirements for consolidating stub generation into a single unified pipeline.

### Pipeline Unification

- [x] **UNIFY-01**: `generate_chemi_stubs()` uses `openapi_to_spec()` instead of `parse_chemi_schemas()`
- [x] **UNIFY-02**: All three generators (ct, chemi, cc) follow identical parsing pattern
- [x] **UNIFY-03**: `ENDPOINT_PATTERNS_TO_EXCLUDE` applied consistently across all schema types
- [x] **UNIFY-04**: Version detection (Swagger 2.0 vs OpenAPI 3.0) works for all schemas

### Code Cleanup

- [x] **CLEAN-01**: Remove or deprecate `parse_chemi_schemas()` if no longer needed
- [x] **CLEAN-02**: Reduce code duplication across the three generator functions

### Validation

- [ ] **VAL-01**: GH Action produces same output as local `Rscript dev/generate_stubs.R` (deferred)
- [x] **VAL-02**: All existing stubs regenerate correctly (no regression)
- [x] **VAL-03**: Chemi stubs benefit from v1.5 Swagger 2.0 body extraction

## Future Requirements

Deferred to future milestone.

### Testing

- **TEST-01**: Unit tests for `extract_body_properties()` (carried from v1.5)
- **TEST-02**: Integration tests for stub generation pipeline (carried from v1.5)

### Advanced Schema Handling

- **ADV-01**: Content-type extraction from `consumes` array (Swagger 2.0)
- **ADV-02**: Response type extraction from `produces` array (Swagger 2.0)
- **ADV-03**: Primitive body types (string, integer without properties)
- **ADV-04**: Array-of-arrays complex nested structures

## Out of Scope

Explicitly excluded for this milestone.

| Feature | Reason |
|---------|--------|
| New endpoint coverage | Focus is consolidation, not expansion |
| GH Action workflow changes | Only ensuring output consistency, not changing workflow |
| Schema download automation | Separate concern from stub generation |
