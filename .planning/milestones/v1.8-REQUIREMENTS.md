# v1.8 Requirements: Testing Infrastructure

**Milestone:** v1.8 Testing Infrastructure
**Created:** 2026-01-30
**Status:** Complete

---

## Goal Statement

Add comprehensive test coverage to the stub generation pipeline (`dev/endpoint_eval/`) to ensure correctness, prevent regressions, and prepare the package for CRAN/rOpenSci submission.

---

## Functional Requirements

### FR-01: Test Infrastructure Setup
- [x] Create `tests/testthat/helper-pipeline.R` with sourcing logic for dev/ files
- [x] Create `tests/testthat/fixtures/schemas/` directory with test schemas
- [x] Add `withr` to Suggests in DESCRIPTION (if not present)
- [x] Implement `.StubGenEnv` cleanup helpers for state isolation

### FR-02: Unit Tests for Pipeline Functions
- [x] `test-pipeline-config.R` — Test `%||%`, `ensure_cols()`, constants
- [x] `test-pipeline-schema-resolution.R` — Test `resolve_schema_ref()`, circular ref handling
- [x] `test-pipeline-openapi-parser.R` — Test `openapi_to_spec()`, `detect_schema_version()`
- [x] `test-pipeline-stub-generation.R` — Test `build_function_stub()`, `is_empty_post_endpoint()`

### FR-03: Integration Tests
- [x] `test-pipeline-integration.R` — End-to-end tests with real schemas
- [x] Verify generated stubs are syntactically valid R code
- [x] Test both Swagger 2.0 and OpenAPI 3.0 schemas

### FR-04: Coverage Verification
- [x] R/ code coverage >= 75% (rOpenSci requirement)
- [x] dev/endpoint_eval/ coverage >= 80% (internal quality target)
- [x] Coverage reporting integrated with Codecov

### FR-05: CI/CD Integration
- [x] Pipeline tests run on PR/merge via GHA
- [x] Coverage badge displays accurate R/ code coverage
- [x] Test failures block PR merge

---

## Non-Functional Requirements

### NFR-01: Performance
- [x] All tests complete in < 60 seconds on GHA runners
- [x] Use `skip_on_cran()` for tests requiring network or long execution

### NFR-02: Maintainability
- [x] Tests follow existing testthat patterns in codebase
- [x] Clear documentation in helper files
- [x] Fixture schemas are minimal but representative

### NFR-03: Reliability
- [x] Tests are deterministic (no flaky tests)
- [x] State cleanup prevents test pollution
- [x] Tests work on Windows, macOS, Linux

## Out of Scope

- Testing wrapper functions in R/ (already have 984 test_that calls)
- Modifying existing vcr cassettes
- Auto-generation of wrapper function tests (future milestone)
- TEST API integration testing (deferred)

---

## Success Criteria

1. All pipeline functions have unit test coverage
2. Integration tests verify end-to-end schema → stub generation
3. R/ coverage >= 75% for rOpenSci submission
4. dev/ coverage >= 80% for internal quality
5. GHA runs tests on every PR/merge
6. No test pollution between test files

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation | Outcome |
|------|------------|--------|------------|---------|
| Global state pollution from `.StubGenEnv` | HIGH | MEDIUM | Use `withr::defer()` for cleanup | MITIGATED |
| Circular ref tests cause infinite loops | MEDIUM | HIGH | Add recursion depth limit in tests | MITIGATED |
| Coverage calculation excludes dev/ | MEDIUM | LOW | Use `file_coverage()` separately | MITIGATED |
| Schema fixtures become outdated | LOW | LOW | Derive from production schemas | ACCEPTED |
