# Milestone v1.0: Stub Generation Fix

**Status:** SHIPPED 2026-01-27
**Phases:** 1-2
**Total Plans:** 4

## Overview

This roadmap delivered a fix for POST endpoint stub generation where simple body schemas (string, array of strings) were not recognized as query parameters. Phase 1 modified the parser and stub builder to handle these simple types correctly, implementing batching and newline collapsing. Phase 2 validated the fix by regenerating stubs and testing against live APIs.

## Phases

**Goal**: Body parameter extraction recognizes simple types and generates correct function signatures
**Depends on**: Nothing (first phase)
**Requirements**: BODY-01, BODY-02, BODY-03, BODY-04, BATCH-01
**Plans**: 2 plans

Plans:
- [x] 01-01-PLAN.md — Extend extract_body_properties() for simple body types (string, string_array)
- [x] 01-02-PLAN.md — Update build_function_stub() for simple body code generation

**Success Criteria (all verified):**
1. `extract_body_properties()` returns structured metadata for string body schemas
2. `extract_body_properties()` returns structured metadata for array-of-strings body schemas

5. Generated POST functions use `Sys.getenv("batch_limit")` instead of hardcoded batch values

## Milestone Summary

**Key Decisions:**
- Use "query" as synthetic parameter name for simple body types (consistent with path parameter pattern)
- Return type="string_array" instead of generic "array" (enables type-specific handling)
- Set required=TRUE for synthetic body parameters (body typically required for POST)
- Add inline schema detection (OpenAPI schemas can be inline without $ref)
- Use three-cassette VCR test strategy (single/multi/invalid for clarity)

**Issues Resolved:**
- POST endpoints with simple body schemas now correctly generate function signatures
- Inline OpenAPI schemas (no $ref) now properly detected and handled

**Technical Debt Incurred:**
- Info: Documentation @description only has lifecycle badge (could add more descriptive text)
- Info: @return is generic (could be more specific about return type)
- Observation: /chemical/search/equal/ endpoint may not be optimal for bulk DTXSID lookups
