# Requirements: ComptoxR v2.2 Package Stabilization

**Defined:** 2026-03-04
**Core Value:** Researchers can query EPA CompTox APIs through stable, well-tested R functions that handle authentication, batching, pagination, and result formatting automatically.

## v2.2 Requirements

### Test Infrastructure

- [ ] **TEST-01**: Test generator correctly detects parameter types from function signatures (no DTXSID passed to limit/search_type/count parameters)
- [ ] **TEST-02**: Test generator reads tidy flag from generic_request() calls and asserts correct return type (tibble for tidy=TRUE, list for tidy=FALSE)
- [ ] **TEST-03**: Pre-existing test failures quarantined with baseline count tracked — CI fails if failure count increases
- [ ] **TEST-04**: All bad VCR cassettes deleted (673 untracked cassettes with wrong parameter values removed)
- [ ] **TEST-05**: Cassette quality gate established — check_cassette_errors() runs before committing new cassettes

### Function Migration — Simple

- [ ] **MIG-01**: All thin wrapper ct_* functions delegate to generated stubs via 1-line call
- [ ] **MIG-02**: All direct template ct_* functions call generic_request() with correct endpoint and projection parameters
- [ ] **MIG-03**: All migrated functions have @lifecycle stable badge in roxygen documentation
- [ ] **MIG-04**: All migrated functions have consistent parameter naming (query for primary identifier)

### Function Migration — Complex

- [ ] **CMIG-01**: ct_bioactivity() dispatcher pattern verified — switch on search_type routes to correct generated stubs, annotate join works
- [ ] **CMIG-02**: ct_lists_all() post-processing verified — conditional projection, coerce/split logic functions correctly
- [ ] **CMIG-03**: ct_demographic_exposure() migrated to generic_request() or generated stub delegation
- [ ] **CMIG-04**: ct_descriptors() migrated to generic_request() or generated stub delegation
- [ ] **CMIG-05**: All complex migrated functions have VCR cassettes covering major code paths

### Build Quality

- [ ] **BLD-01**: R CMD check produces 0 errors on Windows
- [ ] **BLD-02**: R CMD check produces 0 warnings (excluding notes)
- [ ] **BLD-03**: All migrated functions have complete roxygen documentation (@param, @return, @examples)
- [ ] **BLD-04**: devtools::document() generates man pages without errors for all migrated functions

## v2.3 Requirements (Deferred)

### Post-Processing Infrastructure

- **INFRA-01**: Post-processing recipe system designed and implemented (#120)
- **INFRA-02**: Recipe system survives stub regeneration from schema updates

### Advanced Testing

- **ATEST-01**: Snapshot tests for complex function outputs
- **ATEST-02**: All 256 API wrappers have recorded VCR cassettes

## Out of Scope

| Feature | Reason |
|---------|--------|
| Post-processing recipe system (#120) | Deferred — only 3-4 functions need complex post-processing, pattern not yet clear enough to generalize |
| S7 class implementation (#29) | Would break user code, separate concern |
| httptest2 migration | Would require rewriting 706+ cassettes |
| EPI Suite / ECOTOX / GenRA integration | Feature expansion, not stabilization |
| Advanced schema handling (ADV-01-04) | Pipeline work, on hold |
| Session-level result caching | User space concern |
| Cassette re-recording for all 256 wrappers | Requires API key sessions, impractical in v2.2 scope |
