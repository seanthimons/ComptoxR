# Requirements — v1.4 Empty POST Endpoint Detection

## v1.4 Milestone Requirements

### Detection
- [x] **DETECT-01**: Identify POST endpoints with no query parameters — VALIDATED
- [x] **DETECT-02**: Identify POST endpoints with no path parameters — VALIDATED
- [x] **DETECT-03**: Identify POST endpoints with empty/missing body schema (type: object with no properties) — VALIDATED
- [x] **DETECT-04**: Combine all criteria — skip endpoint only when ALL conditions met (POST + no query + no path + empty body) — VALIDATED

### Notification
- [x] **NOTIFY-01**: Warn immediately via `cli::cli_warn()` when skipping an endpoint — VALIDATED
- [x] **NOTIFY-02**: Collect all skipped endpoints during generation run — VALIDATED
- [x] **NOTIFY-03**: Display summary of skipped endpoints at end of generation — VALIDATED

### Scope
- [x] **SCOPE-01**: GET endpoints with no parameters remain valid (not skipped) — VALIDATED
- [x] **SCOPE-02**: POST endpoints with ANY parameter (query, path, or body with properties) are not skipped — VALIDATED

## Future Requirements

None — this is a focused fix for incomplete endpoints.

## Out of Scope

- Handling other incomplete schema patterns (e.g., missing response schemas)
- Automatic endpoint validation beyond empty POST detection
- Adding tests for this feature (deferred to TEST-01/TEST-02 in v2)
