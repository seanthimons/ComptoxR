## Implementation Details

The fix extended `generic_request()` rather than generating standalone code:

1. Added `body_type` parameter to `generic_request()`:
   - `"json"` (default): Uses `httr2::req_body_json()`
   - `"raw_text"`: Uses `httr2::req_body_raw()` with `text/plain`

2. Updated stub generator to detect string body type POST endpoints and generate `body_type = "raw_text"`

3. Generated function inherits all `generic_request()` features:
   - `run_debug` support
   - `run_verbose` support
   - Batching
   - Error handling
   - Response parsing
