**Context:** This was an intentional decision documented in the plan:
> "Do NOT write files to R/ directory during verification. Instead, generate in-memory stubs and compare structure."

**Capability verified:**
- Swagger 2.0 POST endpoints can generate correct stubs
- Body parameters are correctly extracted and included

**Files not written:**

- R/chemi_rdkit_* (POST endpoints)
- R/chemi_mordred_* (POST endpoints)

**Recommendation:** Either write the stub files in a future phase, or update requirements to clarify "capability verification only" was the goal.
