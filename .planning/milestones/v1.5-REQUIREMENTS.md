# Requirements: ComptoxR v1.5 Swagger 2.0 Body Schema Support

**Defined:** 2026-01-29
**Core Value:** Generated API wrapper functions must send requests in the format the API expects

## v1.5 Requirements

Requirements for Swagger 2.0 body schema support. Each maps to roadmap phases.

### Version Detection

- [x] **VERS-01**: Parser detects Swagger 2.0 schemas via `swagger` field at root
- [x] **VERS-02**: Parser detects OpenAPI 3.0 schemas via `openapi` field at root
- [x] **VERS-03**: Version detection routes to appropriate extraction logic

### Body Extraction

- [x] **BODY-01**: Extract body schema from `parameters[].in="body"` for Swagger 2.0
- [x] **BODY-02**: Extract body schema from `requestBody.content` for OpenAPI 3.0 (existing)
- [x] **BODY-03**: Resolve `$ref` to `#/definitions/{SchemaName}` for Swagger 2.0
- [x] **BODY-04**: Handle inline object schemas (no $ref, direct properties)
- [x] **BODY-05**: Validate single body parameter constraint per Swagger 2.0 spec
- [x] **BODY-06**: Validate body/formData mutual exclusivity per spec

### Reference Resolution

- [x] **REF-01**: Fallback chain checks `#/definitions/` then `#/components/schemas/`
- [x] **REF-02**: Version context passed through entire resolution chain
- [x] **REF-03**: Nested `$ref` resolution with configurable depth limit (default: 3)

### Endpoint Filtering

- [x] **FILT-01**: Add "preflight" to ENDPOINT_PATTERNS_TO_EXCLUDE regex

### Integration

- [x] **INTEG-01**: Version detection wired into `openapi_to_spec()` main parser
- [x] **INTEG-02**: All Swagger 2.0 POST endpoints with body schemas generate stubs
- [x] **INTEG-03**: OpenAPI 3.0 stubs unchanged (no regression)
- [x] **INTEG-04**: Regenerate AMOS stubs with correct body parameters (capability verified)
- [x] **INTEG-05**: Regenerate RDKit stubs with correct body parameters (capability verified)
- [x] **INTEG-06**: Regenerate Mordred stubs with correct body parameters (capability verified)

## Future Requirements

Deferred to future milestone. Tracked but not in current roadmap.

### Advanced Schema Handling

- **ADV-01**: Content-type extraction from `consumes` array (Swagger 2.0)
- **ADV-02**: Response type extraction from `produces` array (Swagger 2.0)
- **ADV-03**: Primitive body types (string, integer without properties)
- **ADV-04**: Array-of-arrays complex nested structures

### Testing

- **TEST-01**: Unit tests for `extract_body_properties()` (carried from v1.4)
- **TEST-02**: Integration tests for stub generation pipeline (carried from v1.4)
- **TEST-03**: Cross-version schema equivalence tests

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| formData parameter extraction | Different from body params, file upload use case |
| XML body support | No XML schemas in ComptoxR target APIs |
| Authentication schema parsing | Already handled by generic_request() |
| Response schema stub generation | Focus is request body, not response handling |
