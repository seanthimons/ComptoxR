```
# -- Section 9: Semantic Adjudication Gate ----------------------------------
cli::cli_h2("9. Semantic Adjudication Gate")
```

The section must:

1. Read `lifestage_semantic_adjudication.csv` via the existing `csv_path()` helper.
2. Check required columns:
   `org_lifestage`, `source_ontology`, `source_term_id`, `harmonized_life_stage`, `reproductive_stage`, `route_family`, `adjudication_status`, `adjudication_reason`.
3. Assert allowed `adjudication_status` values:
   `approved_same_semantics`, `approved_exception`, `policy_hold_unresolved`, `needs_context_aware_derivation`, `needs_manual_review`.
4. Assert allowed `adjudication_reason` values:
   `unchanged_harmonized_semantics`, `semantic_change`, `ambiguous_route`, `no_source_backed_candidate`, `broad_source_concept`, `missing_derivation`, `unresolved_policy_tail`.
5. Anti-join resolved baseline terms against adjudication rows by `org_lifestage`; abort if any are missing.
6. Anti-join resolved rows from `lifestage_taxon_intersections.csv` against adjudication rows by `org_lifestage + eco_group + kingdom + class_name`; abort if any are missing.
7. Abort if any resolved lifestage/species-group context is adjudicated as `needs_manual_review`.
8. Warn, not abort, for `needs_context_aware_derivation` rows, because those may be intentional handoff rows for a follow-up phase.
9. Assert that exception rows with changed replacement semantics are not marked `approved_same_semantics`.
10. If example rows such as Sapling are present in the current exception artifact, validate them through the same general semantic-change exception logic used for every other row.
11. Print a compact status/reason/context-scope count table.

Use `cli::cli_abort()` for blocking failures and `cli::cli_alert_success()` for pass messages.
