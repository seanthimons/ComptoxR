<objective>
Create a stable, reviewable semantic adjudication artifact for Phase 36.2 and a repeatable build script that joins baseline source resolution, derivation metadata, curated exceptions, and ECOTOX taxon route context.

Purpose: The current validation proves key coverage, but it does not prove that a source-ID-level derivation is semantically defensible for each reported ECOTOX lifestage/species-group context. This plan makes that judgment explicit without changing runtime ranking or adding ontology sources.

Output: `dev/lifestage/build_semantic_adjudication.R`, `inst/extdata/ecotox/lifestage_semantic_adjudication.csv`, and `dev/lifestage/semantic_derivation_action_report.csv`.
</objective>

  <action>
Create `dev/lifestage/build_semantic_adjudication.R` using the dev script style already used by `build_taxon_router.R` and `validate_36.2.R`.

The script must:

1. Run from project root with `suppressPackageStartupMessages(devtools::load_all(".", quiet = TRUE))`.
2. Read all CSV inputs through a local `csv_path()` helper that first checks `inst/extdata/ecotox/` and then `system.file()`.
3. Filter `lifestage_baseline.csv` to resolved rows only, grouped by `org_lifestage`, taking the rank-1 candidate.
4. Join derivation metadata by `source_ontology + source_term_id`.
5. Join taxon route context from `lifestage_taxon_intersections.csv` and preserve the full composite evidence key: `org_lifestage + eco_group + kingdom + class_name`. Do not collapse the adjudication artifact to dominant route only.
6. Join curated exception metadata by `org_lifestage + source_ontology + source_term_id`.
7. Derive `adjudication_status` and `adjudication_reason` with these rules:
   - If an exception exists and `exception_reason == "semantic_change"`, set `approved_exception` / `semantic_change`.
   - If an exception exists and `exception_reason == "ambiguous_route"`, set `approved_exception` / `ambiguous_route`.
   - If an exception exists and `exception_reason == "no_source_backed_candidate"`, set `approved_exception` / `no_source_backed_candidate`.
   - If derivation fields are missing, set `needs_manual_review` / `missing_derivation`.
   - If the same source key appears with more than one ECOTOX term and exception/replacement evidence shows changed semantics, set `needs_context_aware_derivation` / `broad_source_concept`.
   - Otherwise set `approved_same_semantics` / `unchanged_harmonized_semantics`.
8. Include at minimum these columns in `inst/extdata/ecotox/lifestage_semantic_adjudication.csv`:
   `org_lifestage`, `source_ontology`, `source_term_id`, `source_term_label`, `harmonized_life_stage`, `reproductive_stage`, `derivation_source`, `route_family`, `eco_group`, `kingdom`, `class_name`, `dominant_route`, `taxon_signal_score`, `taxon_signal_share`, `adjudication_status`, `adjudication_reason`, `exception_reason`, `replacement_status`, `replacement_source_ontology`, `replacement_source_term_id`, `replacement_source_term_label`, `replacement_harmonized_life_stage`, `replacement_reproductive_stage`, `context_scope`, `reviewer_notes`.
9. Add a generated `context_scope` value:
   - `dominant_context` for rows where `dominant_route` is true.
   - `non_dominant_same_route_family` for non-dominant rows whose route family matches the dominant route family for that `org_lifestage`.
   - `non_dominant_different_route_family` for non-dominant rows whose route family differs from the dominant route family for that `org_lifestage`.
10. Write all adjudication rows sorted by `org_lifestage`, descending `dominant_route`, descending `taxon_signal_score`, `route_family`, `eco_group`, `kingdom`, `class_name`.
11. Write `dev/lifestage/semantic_derivation_action_report.csv` containing only rows whose status is `needs_context_aware_derivation` or `needs_manual_review`, plus generated rows where `context_scope == "non_dominant_different_route_family"` and the route-family difference may make the lifestage derivation ambiguous.
12. Print counts by `adjudication_status`, `adjudication_reason`, and `context_scope`.

The script must not query live ontology APIs and must not mutate `lifestage_baseline.csv`, `lifestage_derivation.csv`, or runtime resolver code.
  </action>
