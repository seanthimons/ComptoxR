# Phase 36.2: Dictionary Rebuild Validation - Context

**Gathered:** 2026-04-27
**Status:** Ready for planning

<domain>
## Phase Boundary

Validate the regenerated source-backed lifestage dictionary artifacts after the unresolved coverage audit, with the active focus narrowed to semantic adjudication: for each resolved ontology/source concept, determine whether its derived `harmonized_life_stage` and `reproductive_stage` are defensible for each ECOTOX `org_lifestage` and taxonomic/context route where it is used.

This phase is not an ontology expansion phase. Deterministic routing and evidence infrastructure are checkpointed in commit `2c9d1e9`; the remaining risk is semantic overconfidence, not missing source coverage.

</domain>

<decisions>
## Implementation Decisions

### Semantic adjudication replaces ontology expansion as the next step
- **D-01:** Do not continue adding ontologies as the default next move. Additional ontology sources are allowed only after gap analysis proves that no source-backed candidate exists for a specific ECOTOX term/context.
- **D-02:** Treat ECOTOX as the vocabulary being harmonized. Ontology evidence selects candidate source concepts, but it does not by itself define the final harmonized category for every ECOTOX context.
- **D-03:** The key question for each resolved mapping is: "Is this derived harmonized category defensible for this ECOTOX term/context?"

### Evidence, ranking, and human judgment split
- **D-04:** Ontology/provider evidence should select candidate source concepts.
- **D-05:** Deterministic policy should rank candidates by route-family ontology priority, candidate score, and stable tie-breakers.
- **D-06:** Human judgment should be limited to small, explicit, reviewable decisions: documented exceptions, source-ID derivation rules, or broad context-aware derivation rules.
- **D-07:** Success is not "zero human decisions." Success is that every human decision is explicit, reviewable, source-backed where possible, and small enough to audit.

### Derivation model must be context-aware
- **D-08:** `lifestage_derivation.csv` is the structural weak point for this phase because it maps `source_ontology + source_term_id` directly to derived fields. Planning must test whether that key is too coarse for broad concepts.
- **D-09:** Broad ontology concepts such as `PO:0007134` must not be treated as globally sufficient evidence for a single harmonized category without checking the ECOTOX term/context. `Sapling` is a useful example of the failure mode, not an iron-clad policy row that must remain unchanged forever.
- **D-10:** If one source concept can imply different harmonized categories depending on ECOTOX term, route family, or taxonomic context, the planner should design an adjudication artifact that can represent that distinction instead of forcing a global source-ID derivation.

### Confidence semantics
- **D-11:** The bigger risk is wrong confidence. Validation should identify mappings that look resolved because an ontology candidate exists but whose derived harmonized category is not semantically defensible.
- **D-12:** A candidate should not be promoted just because it is source-backed and high scoring. Promotion requires unchanged or explicitly justified harmonized semantics.
- **D-13:** Curated exceptions should remain minimal, but minimization cannot mean deleting human judgment where the replacement changes meaning.

### Exception and gap policy
- **D-14:** Retained curated rows should be explainable by one of the existing reason families: `semantic_change`, `ambiguous_route`, or `no_source_backed_candidate`.
- **D-15:** `no_source_backed_candidate` should be assigned only after gap analysis checks the current source-backed candidates and shows that none are defensible.
- **D-16:** The unresolved tail remains policy-held unless case-by-case gap analysis proves a source-backed promotion is warranted.

### Validation expectations
- **D-17:** The semantic adjudication artifact should make every resolved baseline source key reviewable against ECOTOX `org_lifestage`, route/taxon context, chosen source concept, current derived fields, and any competing replacement candidate.
- **D-20:** Semantic adjudication must operate on the reported ECOTOX lifestage paired with its reported species/taxon group evidence (`org_lifestage + eco_group + kingdom + class_name`), not only the lifestage label in isolation. The 1000+ taxon intersection rows are generated review evidence; humans should only touch the exception/action queue.
- **D-18:** CI/dev validation should fail or flag when a resolved baseline mapping lacks explicit adjudication, when a promoted replacement changes derived semantics without an approved exception, or when an exception lacks one of the accepted reason codes.
- **D-19:** Preserve the existing targeted verification commands after semantic changes:
  - `Rscript -e "devtools::test(filter='eco_lifestage_data')"`
  - `Rscript dev/lifestage/validate_36.2.R`

### Agent's Discretion
- Exact file format for the semantic adjudication artifact, as long as it is diffable, reviewable, and can be enforced by testthat/dev validation.
- Exact column names for additional context fields, as long as downstream reviewers can see ECOTOX term, route/taxon context, current source concept, derived fields, candidate replacement, reason code, and adjudication status.
- Whether to extend `lifestage_derivation.csv` directly or introduce a separate adjudication artifact first, provided stable package data only changes after review.

</decisions>

<specifics>
## Specific Ideas

- The original hand-annotated approach was probably better in some cases because a human silently supplied domain judgment. The current approach is worse when it pretends ontology lookup can replace that judgment.
- The viable middle ground is: ontology evidence selects candidates, deterministic policy ranks them, and humans adjudicate only small documented exceptions or broad derivation rules.
- `Sapling` is a useful example case. Current curated mapping: `S11:S1127 -> Juvenile / FALSE`. Candidate replacement: `PO:0007134 -> Adult / FALSE`. Current reason: `semantic_change`. This illustrates the broader rule that changed derived semantics need explicit adjudication.
- Commit `2c9d1e9` is the deterministic routing checkpoint. Do not reopen route-family ranking as the main problem unless a new failing case proves the checkpoint policy itself is wrong.

</specifics>

## Existing Code Insights

### Reusable Assets
- `.eco_lifestage_route_ontology_priority_policy()` and `.eco_lifestage_candidate_ontology_priority()` load and apply the route-family priority CSV.
- `.eco_lifestage_taxon_route_lookup()` prefers the full `lifestage_taxon_intersections.csv` artifact and falls back to dominant profiles only when needed.
- `.eco_lifestage_rank_candidates()` already has access to detected domains and taxon route family before ranking candidates.
- `.eco_lifestage_curated_candidates()` provides a constrained way to inject source-backed curated bridges without making all curation ordinary path behavior.
- `.eco_lifestage_derivation_coverage_report()` and `dev/lifestage/validate_36.2.R` already provide a structure for turning mapping gaps into a reviewable report.
- `tests/testthat/test-eco_lifestage_data.R` already contains current Sapling example checks and the curated minimization invariant; Phase 36.2 should generalize this into semantic-change adjudication rather than treating Sapling as a special permanent rule.

### Established Patterns
- Data policies live in committed CSVs under `inst/extdata/ecotox/` when they need to be reviewable.
- Phase validation lives in `dev/lifestage/validate_*.R`.
- Permanent gates for CSV artifacts live in `tests/testthat/test-eco_lifestage_data.R`.
- Runtime warnings use `cli::cli_warn()` for non-fatal unresolved/audit issues; CI/dev validation can be stricter.
- Source-backed canonical rows enter `lifestage_dictionary`; ambiguous, unresolved, or underived rows are quarantined/reviewed instead of silently promoted.

### Integration Points
- New adjudication/report artifact should be produced from `lifestage_baseline.csv`, `lifestage_derivation.csv`, `lifestage_curated_exceptions.csv`, and `lifestage_taxon_intersections.csv`.
- The adjudication artifact should preserve the reported lifestage/species-group pairing from `lifestage_taxon_intersections.csv` with a composite key of `org_lifestage`, `eco_group`, `kingdom`, and `class_name`.
- Validation should extend `dev/lifestage/validate_36.2.R` and `tests/testthat/test-eco_lifestage_data.R`.
- If source-ID-only derivation is insufficient, planners should route through a reviewed context-aware artifact rather than mutating resolver ranking.

- Derivation map: 96 rows across `manual_curator`, `permanently_unresolved`, `manual_derivation`, `auto_ontology_category_map`, and `baseline_curated_source_id`.
- Curated exceptions: 24 rows; exception reasons are already constrained to `semantic_change`, `ambiguous_route`, and `no_source_backed_candidate`.
- Taxon intersections: 1004 rows available for ECOTOX term/context adjudication.
