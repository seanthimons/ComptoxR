Create 4 disposable probe scripts that test external data sources (broader OLS4, BioPortal Annotator, Wikidata SPARQL, FAO/AGROVOC) against representative unresolved lifestage terms to determine which sources can programmatically resolve alias/synonym terms.

Purpose: Probe results inform Phase 36.2 adapter decisions -- which sources become permanent adapters and which terms each source covers. These scripts are exploratory dev tools, not production code.

Output: 4 probe scripts at dev/lifestage/probe_*.R, each printing structured results showing term coverage per source.

**probe_ols4.R** -- Broader OLS4 search that removes the ontology restriction and adds queryFields:

1. Script header: shebang, comment, `devtools::load_all()` per existing pattern
2. Define test terms per D-05 -- two groups:
   - Simple: `c("Fry", "Fingerling", "Alevin", "Yearling", "Pullet", "Weanling")`
   - Complex: `c("Copepodid", "Glochidia", "Trophozoite", "Naiad", "Sporeling", "Zygospore")`
   - Agricultural: `c("Heading", "Jointing", "Tiller stage", "Sapling")`
3. Define relevant ontology prefixes for post-filtering: `c("UBERON:", "PO:", "ECOCORE:", "BTO:", "EFO:", "ZFA:", "FBdv:", "MeSH:")`
4. For each term, make a broader OLS4 search request:
   ```r
   httr2::request("https://www.ebi.ac.uk/ols4/api/search") |>
     httr2::req_url_query(
       q = term,
       queryFields = "label,synonym,description",
       rows = 25
     )
   ```
5. Parse response, extract `response$docs` data frame
6. Post-filter docs by `obo_id` prefix matching the relevant prefixes list
7. For each term, print: term name, total raw hits, filtered hits count, top 3 filtered results showing `obo_id`, `label`, `ontology_name`, and whether the match was in label/synonym/description
8. Wrap each API call in `tryCatch()` with `cli::cli_warn()` on failure
9. Add `Sys.sleep(0.3)` between terms to be respectful
10. At the end, print a summary table: term | found (yes/no) | best ontology | best obo_id

**probe_agrovoc.R** -- FAO/AGROVOC REST search:

1. Script header per existing pattern
2. Define test terms per D-14:
   - Agricultural: `c("Fry", "Fingerling", "Heading", "Jointing", "Tiller stage", "Sapling", "Rootstock", "Rooted cuttings")`
   - Also try alternate spellings: `c("tillering", "cuttings")`
3. For each term, two-step fetch:
   - Step 1: Search via `https://agrovoc.fao.org/browse/rest/v1/search/` with `query = term, lang = "en"`

4. Extract concept label, URI, and any broader/narrower terms from detail response
5. Print per-term results: term | found | concept_uri | preferred_label
6. Wrap in tryCatch, add Sys.sleep(0.5) between terms
7. Print summary table at end

**probe_wikidata.R** -- Wikidata SPARQL for UBERON cross-references:

1. Script header per existing pattern
2. Define test terms per D-05/D-14:
   - Simple: `c("fry", "fingerling", "alevin", "yearling", "weanling")`
   - Complex: `c("copepodid", "glochidia", "naiad", "trophozoite")`
3. For each term, construct SPARQL query using `glue::glue()`:
   ```sparql
   SELECT ?item ?itemLabel ?uberonId WHERE {
     ?item wdt:P1554 ?uberonId .
     ?item rdfs:label ?itemLabel .
     FILTER(LANG(?itemLabel) = "en")
     FILTER(CONTAINS(LCASE(?itemLabel), "{tolower(term)}"))
   }
   LIMIT 10
   ```
4. Send query via GET to `https://query.wikidata.org/sparql` with:
   - `query` as URL parameter
   - `Accept: application/sparql-results+json` header
   - `User-Agent: ComptoxR/dev probe script (verfassergeist@gmail.com)` header (required by Wikidata)
5. Parse response, extract item labels and UBERON IDs
6. Also try a second query pattern searching for `wdt:P3841` (Animal Diversity Web ID) for broader coverage
7. Print per-term: term | wikidata_hits | uberon_ids_found
8. Wrap in tryCatch, add `Sys.sleep(1)` between terms (Wikidata rate limiting is stricter)
9. Summary table at end

Both scripts use `cli::cli_h1()`, `cli::cli_h2()`, `cli::cli_alert_success()`, `cli::cli_alert_danger()` for structured output.
