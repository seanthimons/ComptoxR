# Phase 36.1: Unresolved Coverage Audit - Research

**Researched:** 2026-04-23
**Domain:** Ontology term classification, external vocabulary API prototyping, CSV audit artifact
**Confidence:** HIGH

## Summary

Phase 36.1 classifies all 69 unresolved lifestage terms from `lifestage_baseline.csv` into 5 triage buckets, prototypes 4 external data sources (broader OLS4, BioPortal Annotator, Wikidata SPARQL, FAO/AGROVOC REST) to determine which can programmatically resolve alias/synonym terms, and produces a committed audit CSV at `inst/extdata/ecotox/lifestage_audit.csv`. The phase also adds 4 administrative term rows to `lifestage_derivation.csv` and wires a `cli_warn` gate into the rebuild path for unclassified terms.

Research confirms all 4 external APIs are accessible without authentication barriers (OLS4 and AGROVOC are fully open; BioPortal requires a free API key; Wikidata SPARQL is open). Live API testing during research validated that broader OLS4 search (removing ontology restriction, adding `queryFields=label,synonym,description`) dramatically increases coverage -- terms like "naiad", "trophozoite", and "copepodid" that fail current restricted search are found in ECOCORE, EFO, MeSH, and as narrow_synonyms of UBERON terms. AGROVOC covers agricultural terms ("fry", "heading") but has gaps ("jointing", "sapling", "tiller" return empty).

**Primary recommendation:** Execute triage classification first (pure domain knowledge task requiring no API calls), then run probe scripts sequentially against OLS4-broader > BioPortal > Wikidata > AGROVOC, populating the `candidate_source` column in the audit CSV with results.

### Pattern 1: Broader OLS4 Search
**What:** Remove ontology restriction and add `queryFields=label,synonym,description` to search across all OLS4 ontologies, then post-filter by relevance.
**When to use:** For terms that fail the current UBERON/PO-restricted search.
**Example:**
```r
# Source: Live API testing during research (2026-04-23)
# Current restricted search (misses fish/invertebrate terms):
httr2::request("https://www.ebi.ac.uk/ols4/api/search") |>
  httr2::req_url_query(q = "copepodid", ontology = "uberon", rows = 25)
# Returns: 0 results

# Broader search (finds ECOCORE hits):
httr2::request("https://www.ebi.ac.uk/ols4/api/search") |>
  httr2::req_url_query(
    q = "copepodid",
    queryFields = "label,synonym,description",
    rows = 25
  )
# Returns: 5 results in ECOCORE (copepodid stages 2-6)
```

### Pattern 3: BioPortal Annotator
**What:** Submit term text to BioPortal Annotator, which searches across 900+ ontologies.
**When to use:** For specialist terms not in OLS4 (glochidia, trophozoite as life stage).
**Example:**
```r
# Source: BioPortal API documentation [CITED: data.bioontology.org/documentation]
# Requires free API key from bioportal.bioontology.org
httr2::request("https://data.bioontology.org/annotator") |>
  httr2::req_url_query(
    text = "glochidia",
    ontologies = "UBERON,ZFA,FBdv,ECOCORE",
    apikey = Sys.getenv("BIOPORTAL_API_KEY")
  ) |>
  httr2::req_perform() |>
  httr2::resp_body_json()
```

### Pattern 4: Wikidata SPARQL
**What:** Query Wikidata for concepts with skos:exactMatch/closeMatch to UBERON/PO.
**When to use:** For terms with cross-ontology mappings not discoverable via OLS4.
**Example:**
```r
# Source: Wikidata SPARQL tutorial [CITED: wikidata.org/wiki/Wikidata:SPARQL_tutorial]
query <- '
SELECT ?item ?itemLabel ?uberonId WHERE {
  ?item wdt:P1554 ?uberonId .
  ?item rdfs:label ?itemLabel .
  FILTER(LANG(?itemLabel) = "en")
  FILTER(CONTAINS(LCASE(?itemLabel), "fry"))
}
LIMIT 10
'
httr2::request("https://query.wikidata.org/sparql") |>
  httr2::req_url_query(query = query) |>
  httr2::req_headers(Accept = "application/sparql-results+json") |>
  httr2::req_headers(`User-Agent` = "ComptoxR/dev probe script") |>
  httr2::req_perform() |>
  httr2::resp_body_json()
```

### Anti-Patterns to Avoid
- **Searching OLS4 with ontology restriction only:** The current `.eco_lifestage_query_ols4()` restricts to UBERON and PO, missing terms in ECOCORE, BTO, EFO, FBdv, ZFA, and MeSH. Probe scripts must search broadly.
- **Assuming AGROVOC covers all agricultural terms:** Research showed "jointing", "sapling", and "tiller" return empty from AGROVOC despite being agricultural concepts.
- **Hitting BioPortal without rate limiting:** BioPortal receives 100M API calls/month; be respectful with `Sys.sleep(0.5)` between calls in probe scripts.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Synonym matching | Custom synonym database | OLS4 `exact_synonyms` + `narrow_synonyms` fields | OLS4 already indexes synonym data from source ontologies |
| Cross-ontology discovery | Manual ontology-by-ontology search | BioPortal Annotator (searches 900+ ontologies) | BioPortal's coverage is unmatched for specialist terms |
| Agricultural vocabulary | Custom crop stage mapping | AGROVOC REST API | FAO maintains authoritative agricultural terminology |
| SPARQL query building | String concatenation | glue templates with parameterized terms | Prevents injection and improves readability |

**Key insight:** The 69 unresolved terms span 6+ biological domains (fish, amphibian, invertebrate, microbial, agricultural, administrative). No single ontology covers them all. The probe scripts' value is determining which source covers which domain, not building a universal resolver.

## Common Pitfalls

### Pitfall 1: OLS4 Returns Cross-Ontology Noise
**What goes wrong:** Searching OLS4 without ontology restriction returns gene/protein results (e.g., "fry" returns 117 hits, mostly for the FRY gene in fruit fly and human).
**Why it happens:** OLS4 indexes 250+ ontologies; many terms are polysemous.
**How to avoid:** Post-filter by relevant ontology prefixes (UBERON, PO, ECOCORE, BTO, EFO, ZFA, FBdv, MeSH) or by entity type (class, not individual).
**Warning signs:** Result count > 50 for a simple term, results from PR/CCO/GeXO ontologies.

### Pitfall 2: BioPortal API Key Not Set
**What goes wrong:** BioPortal returns 403 Forbidden without an API key.
**Why it happens:** Unlike OLS4 and AGROVOC, BioPortal requires authentication.
**How to avoid:** Probe script should check `Sys.getenv("BIOPORTAL_API_KEY")` at startup and skip with informative message if empty. Key is free from bioportal.bioontology.org.
**Warning signs:** HTTP 403 responses.

### Pitfall 3: AGROVOC Empty Results for Valid Agricultural Terms
**What goes wrong:** Terms like "jointing", "sapling", "tiller" return empty despite being real agricultural concepts.
**Why it happens:** AGROVOC uses specific preferred labels that may differ from ECOTOX terminology.
**How to avoid:** Try alternate spellings and related terms (e.g., "tillering" instead of "tiller").
**Warning signs:** Empty results for terms you expect AGROVOC to cover.

### Pitfall 4: Wikidata Rate Limiting
**What goes wrong:** Wikidata SPARQL endpoint returns 429 Too Many Requests.
**Why it happens:** Wikidata enforces rate limits on SPARQL queries.
**How to avoid:** Include `User-Agent` header, batch queries, add `Sys.sleep(1)` between calls.
**Warning signs:** HTTP 429 responses or connection timeouts.

### Pitfall 5: Conflating "Not Found" with "Doesn't Exist"
**What goes wrong:** Classifying a term as `out_of_scope_biology` because one API didn't find it, when another API would.
**Why it happens:** Different ontologies have different coverage.
**How to avoid:** Test each term against ALL 4 sources before final classification. The probe scripts exist precisely for this purpose.
**Warning signs:** A term classified as out-of-scope that a domain expert would recognize as a valid life stage.

### CI Anti-Join Test
```r
# Source: CONTEXT.md D-12, pattern from existing test-eco_lifestage_data.R
test_that("every unresolved baseline term has an audit classification", {
  testthat::skip_if_not_installed("readr")
  baseline_path <- system.file("extdata", "ecotox", "lifestage_baseline.csv", package = "ComptoxR")
  audit_path <- system.file("extdata", "ecotox", "lifestage_audit.csv", package = "ComptoxR")
  testthat::skip_if(nchar(baseline_path) == 0 || nchar(audit_path) == 0)
  baseline <- readr::read_csv(baseline_path, show_col_types = FALSE)
  audit <- readr::read_csv(audit_path, show_col_types = FALSE)
  unresolved <- dplyr::filter(baseline, source_match_status == "unresolved")
  gaps <- dplyr::anti_join(unresolved, audit, by = "org_lifestage")
  testthat::expect_equal(nrow(gaps), 0L)
})
```

   - What we know: OLS4 broader search found copepodid stages 2-6 in ECOCORE
   - What's unclear: Whether ECOCORE is stable/maintained enough for production use in Phase 36.2
