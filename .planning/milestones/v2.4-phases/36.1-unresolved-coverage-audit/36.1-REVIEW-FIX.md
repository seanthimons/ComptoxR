## Fixed Issues

### WR-01: `!nrow(accepted)` used as boolean -- falsy-zero anti-pattern

**Files modified:** `R/eco_lifestage_patch.R`
**Commit:** eaf1a07
**Applied fix:** Replaced `if (!nrow(accepted))` with `if (nrow(accepted) == 0L)` at line 691. The idiomatic explicit comparison avoids the edge case where `nrow(NULL)` returns `NULL` and `!NULL` throws an error, rather than relying on integer-to-logical coercion of zero.

### WR-02: `baseline_matches` scalar/vector mismatch with `identical()`

**Files modified:** `R/eco_lifestage_patch.R`
**Commit:** c84734b
**Applied fix:** Added a guard that calls `cli::cli_abort()` if the baseline CSV contains rows from multiple ECOTOX releases (length > 1), which would indicate a corrupt or mis-generated baseline. Updated the `baseline_matches` assignment to use `length(baseline_releases) == 1L && identical(baseline_releases, ecotox_release)` to make the scalar expectation explicit and prevent silent mismatch when the vector has length > 1. Status: fixed: requires human verification (logic change).

### WR-03: Audit gate uses `system.file()` without a dev-path fallback

**Files modified:** `R/eco_lifestage_patch.R`
**Commit:** af83ca6
**Applied fix:** Added a dev-path fallback (`file.path("inst", "extdata", "ecotox", "lifestage_audit.csv")`) after the `system.file()` call in the audit coverage gate, consistent with the existing pattern used by `.eco_lifestage_baseline_path()` and `.eco_lifestage_derivation_path()`. This ensures the audit gate is active during `devtools::load_all()` development sessions.

### WR-04: `strict_seed` flag may incorrectly evaluate to `TRUE` in auto mode

**Files modified:** `R/eco_lifestage_patch.R`
**Commit:** a15c6c3
**Applied fix:** Added `!isTRUE(force)` as a third condition in the `strict_seed` assignment. When `force = TRUE`, the strict seed guard is now disabled, allowing live resolution fallback even when the outer `refresh` parameter is `"cache"` or `"baseline"`. Without this, `force = TRUE` with `refresh = "cache"` would still abort on missing terms instead of falling back to live resolution. Status: fixed: requires human verification (logic change).
