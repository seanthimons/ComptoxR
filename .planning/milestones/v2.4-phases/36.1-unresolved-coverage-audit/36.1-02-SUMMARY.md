# Phase 36.1 Plan 02: Unresolved Coverage Audit Probe Scripts Summary

**4 external ontology probe scripts (OLS4 broader search, BioPortal Annotator, Wikidata SPARQL, AGROVOC REST) covering both simple and complex unresolved lifestage terms with rate limiting, error handling, and structured output for Phase 36.2 adapter decisions**

- **Completed:** 2026-04-23T20:29:15Z

## Accomplishments

- Created `probe_ols4.R`: removes UBERON/PO ontology restriction, adds `queryFields=label,synonym,description`, post-filters by 8 relevant prefixes, tests 16 terms (6 simple, 6 complex, 4 agricultural)
- Created `probe_agrovoc.R`: two-step FAO/AGROVOC REST search (search then concept detail), tests 8 primary terms plus 2 alternate spellings, documents known empty-result pitfalls
- Created `probe_bioportal.R`: BioPortal Annotator across 7 ontologies (UBERON, ZFA, FBdv, ECOCORE, BTO, EFO, MeSH), gracefully exits when `BIOPORTAL_API_KEY` is missing, tests 11 terms including out-of-scope checks
- Created `probe_wikidata.R`: dual SPARQL pattern (P1554 UBERON cross-ref + P3841 ADW ID), mandatory User-Agent header, glue-based query parameterization, `Sys.sleep(1)` between terms

## Decisions Made

- BioPortal probe uses `quit(save = "no", status = 0)` (not `cli::cli_abort`) when API key is missing — disposable probe scripts must not fail CI or block the team if key is unavailable
- OLS4 post-filter uses `startsWith(obo_id, RELEVANT_PREFIXES)` rather than ontology param to allow broad initial search while eliminating gene/protein noise (Pitfall 1 from RESEARCH.md)
- Wikidata tests both P1554 and P3841 because assumption A2 (P1554 = UBERON) may not cover all life stage concepts; P3841 provides a second cross-reference path
- AGROVOC probe documents known gaps ("jointing", "sapling", "tiller") in script output per Pitfall 3 from RESEARCH.md
