<threat_model>
## Trust Boundaries

This phase operates on local package code, local DuckDB metadata tables, Plumber JSON responses, and generated documentation. It does not call external ontology providers or handle credentials. The main risks are stale local database schemas, accidental exposure of internal provenance columns in default output, and runtime use of quarantine review rows.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-01 | Integrity | Stale ECOTOX DuckDB schema | mitigate | Add local schema guard for required lifestage tables/columns and abort with patch/rebuild guidance |
| T-38-02 | Information Disclosure | Default `eco_results()` output | mitigate | Hide join/debug/provenance fields by default; expose them only with `lifestage_details = TRUE` |
| T-38-03 | Integrity | Runtime lifestage join boundary | mitigate | Test/source-check that `.eco_enrich_metadata()` never references `lifestage_review` |
| T-38-04 | Repudiation | Public API documentation | mitigate | Regenerate `man/eco_results.Rd` so docs match the compact/detailed contract |
</threat_model>
