# Roadmap: ComptoxR v2.2 Package Stabilization

## Overview

This milestone migrates all user-facing ct_* functions to use centralized request templates, establishing a stable API surface. The journey proceeds through four phases: fix test infrastructure blockers, migrate simple thin wrapper functions, migrate direct template functions, and validate build quality. Success means R CMD check passes cleanly, all migrated functions carry lifecycle badges, and researchers have predictable APIs that survive schema updates.

## Phases

**Phase Numbering:**
- Phases 27-30: v2.2 Package Stabilization work
- Previous phases (1-26): See .planning/MILESTONES.md

- [ ] **Phase 27: Test Infrastructure Stabilization** - Fix test generator bugs, establish cassette quality gates, quarantine broken tests
- [ ] **Phase 28: Thin Wrapper Migration** - Migrate ~21 functions to 1-line stub delegation pattern with lifecycle badges
- [ ] **Phase 29: Direct Template Migration** - Migrate ~7 functions to generic_request() pattern with projection parameters
- [ ] **Phase 30: Build Quality Validation** - Complete documentation, verify R CMD check passes, validate all success criteria

## Phase Details

### Phase 27: Test Infrastructure Stabilization
**Goal**: Test infrastructure enables confident migration with reliable signal (no poisoned cassettes, correct parameter types, tracked failure baseline)
**Depends on**: Phase 26 (v2.1 Test Infrastructure)
**Requirements**: TEST-01, TEST-02, TEST-03, TEST-04, TEST-05
**Success Criteria** (what must be TRUE):
  1. Test generator detects parameter types correctly — no more DTXSID passed to limit/count/search_type parameters
  2. Test generator reads tidy flag from generic_request() calls — tibble vs list assertions match actual return types
  3. All 673 bad cassettes deleted — no poisoned HTTP error responses in test fixtures
  4. Cassette quality gate enforced — check_cassette_errors() runs before commits, rejects cassettes with 4xx/5xx responses
  5. Baseline failure count tracked — CI fails if test failures increase beyond 297 pre-existing failures

### Phase 28: Thin Wrapper Migration

**Depends on**: Phase 27

**Success Criteria** (what must be TRUE):

  2. All migrated functions have @lifecycle stable badge in roxygen documentation
  3. All migrated functions use consistent parameter naming (query for primary identifier)

### Phase 30: Build Quality Validation
**Goal**: Package passes R CMD check cleanly on Windows with complete documentation and verified complex functions
**Depends on**: Phase 29
**Requirements**: CMIG-01, CMIG-02, CMIG-05, BLD-01, BLD-02, BLD-03, BLD-04
