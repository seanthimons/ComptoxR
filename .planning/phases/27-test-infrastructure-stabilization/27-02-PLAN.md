<task type="auto">
  <name>Task 2: Create parallel cassette recording script with mirai</name>
  <files>dev/record_cassettes.R</files>
  <action>
Create `dev/record_cassettes.R` -- a fire-and-forget script for parallel cassette re-recording using mirai. Per user decisions:

1. **Pre-flight checks:**
   - Verify `ctx_api_key` environment variable is set and non-empty. Abort with helpful message if missing.
   - Verify mirai package is available. Abort if not installed.
   - Verify `tests/testthat/fixtures/` directory exists.

2. **Major family discovery:**
   - List all .yml files in `tests/testthat/fixtures/`
   - Extract major family from filename convention: split on `_`, take first two elements (e.g., `ct_bioactivity`, `chemi_amos`)
   - Group cassettes by major family
   - Report family counts before starting

3. **Cassette-to-test-file mapping:**
   - For each cassette file, find the test file that uses it by:
     a. Read the cassette filename (e.g., `ct_hazard_cancer_search_single.yml`)
     b. Search test files for `use_cassette("ct_hazard_cancer_search_single"` pattern
     c. Cache the mapping to avoid re-scanning
   - If no test file found for a cassette, log warning and skip it

4. **Parallel recording with mirai:**
   - `daemons(n = 4)` -- 4 workers (conservative to avoid rate limiting; EPA APIs are government servers)
   - Process one major family at a time (sequential across families, parallel within family)
   - For each family:
     a. Delete existing cassettes for that family
     b. Use `mirai_map()` to run test files in parallel (each test file records its cassettes via VCR)
     c. Each mirai task: `testthat::test_file(test_file, reporter = "minimal")` wrapped in tryCatch
     d. Collect results (success/failure per cassette)

5. **Failure threshold:**
   - After each family completes, calculate failure rate
   - If failure rate > 15% for that family, halt the entire run
   - Log message: "Failure rate X% exceeds 15% threshold -- likely systemic issue (expired key, API outage)"

6. **Logging and progress:**
   - Use cli package for all output
   - Log: family start, cassette count per family, failures per family, overall progress
   - At end: summary table of families with pass/fail counts
   - Write a log file to `dev/reports/cassette_recording_log.txt` with timestamped results

7. **Cleanup:**
   - `daemons(0)` to shut down workers
   - Run `dev/check_cassette_health.R` automatically after recording completes (if no threshold halt)

8. **Script structure:**
   - Hashbang: `#!/usr/bin/env Rscript`
   - Uses `here::here()` for all paths
   - Does NOT source package code or helper-vcr.R at top level (test files handle their own VCR setup)
   - Self-contained -- user runs: `Rscript dev/record_cassettes.R`

NOTE: Use `mirai::daemons()` and `mirai::mirai_map()` -- NOT `mirai::mirai()` in a loop. The `mirai_map()` function handles collection automatically.

NOTE: The test files already have `vcr::use_cassette()` calls that record cassettes when no cassette file exists. So the recording strategy is: delete old cassette -> run test file -> VCR records new cassette from live API.
