<objective>
Add pagination detection to the stub generation pipeline so that `openapi_to_spec()` classifies each endpoint's pagination strategy and stores metadata in the spec tibble.

Purpose: This is the foundation for auto-pagination (Phase 20) and paginated stub generation (Phase 21). Without detection metadata, downstream phases cannot know which endpoints are paginated or how they paginate.

Output: `PAGINATION_REGISTRY` constant in `00_config.R`, `detect_pagination()` function in `04_openapi_parser.R`, two new columns (`pagination_strategy`, `pagination_metadata`) in the spec tibble, default columns in `07_stub_generation.R`, and tests.
</objective>

**In `dev/endpoint_eval/04_openapi_parser.R`:**

Add `detect_pagination()` function in the helpers section (after `order_path_by_route()`, before the `# Main Parsing Functions` comment, around line 309). The function:

1. Takes: `route`, `path_params`, `query_params`, `body_params` (all comma-separated strings matching the tibble format), and `registry = PAGINATION_REGISTRY`.
2. Splits each param string into character vectors using `strsplit(x, ",")` with `trimws()`.
3. Iterates through registry entries in order (most specific first since route-pattern entries come first).
4. For each entry:
   - If `route_pattern` is not NULL, check `stringr::str_detect(route, entry$route_pattern)`. If no match, skip (`next`).
   - For the cursor strategy specifically: check that "limit" is in path_vec AND "cursor" is in query_vec.
   - For entries with `param_location = "path"` (non-cursor): check `all(entry$param_names %in% path_vec)`.
   - For entries with `param_location = "query"`: check `all(entry$param_names %in% query_vec)`.
   - For entries with `param_location = "body"`: check `all(entry$param_names %in% body_vec)`.
5. On first match, return `list(strategy = entry$strategy, registry_key = entry_name, params = entry$param_names, param_location = entry$param_location, description = entry$description)`.
6. If no match, return `list(strategy = "none", registry_key = NA_character_, params = character(0), param_location = NA_character_, description = "No pagination detected")`.

Use `%||%` for null-coalescing (already defined in 00_config.R). Use `nzchar()` to guard against empty strings.

IMPORTANT: Do NOT use `stringr::str_detect()` on NULL route patterns -- the `if (!is.null(entry$route_pattern))` guard must come first. Follow the exact code pattern from RESEARCH.md lines 274-328.
