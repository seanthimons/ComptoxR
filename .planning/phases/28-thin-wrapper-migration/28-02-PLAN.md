```r
ct_lists_all <- function(return_dtxsid = FALSE, coerce = FALSE) {
  projection <- if (!return_dtxsid) "chemicallistall" else "chemicallistwithdtxsids"
  df <- ct_chemical_list_all(projection = projection)
  cli::cli_alert_success("{nrow(df)} lists found!")
  if (return_dtxsid & coerce) {
    cli::cli_alert_warning("Coercing DTXSID strings per list to list-column!")
    df <- df %>% split(.$listName) %>% purrr::map(., as.list) %>%
      purrr::map(., ~ { .x$dtxsids <- stringr::str_split(.x$dtxsids, pattern = ",") %>% purrr::pluck(1); .x })
  } else if (!return_dtxsid & coerce) {
    cli::cli_alert_warning("You need to request DTXSIDs to coerce!")
  }
  return(df)
}
```

- `validate_similarity(data)` (@noRd): Check data$params$similarity is numeric and between 0-1. Use cli::cli_abort() for errors. Return data unchanged if valid. This is a pre_request hook.

**R/hooks/list_hooks.R:**
- `uppercase_query(data)` (@noRd): Pre-request hook. Uppercases data$params$query via toupper() (no stringr needed for simple case). Return modified data.

- `extract_dtxsids_if_requested(data)` (@noRd): Post-response hook. If data$params$extract_dtxsids is TRUE, extract DTXSIDs from data$result. Handle duplicate names (multiple results) by collecting all dtxsids fields. Split comma-separated strings, deduplicate, return character vector. If FALSE, return data$result as-is. Extract logic from ct_list.R lines 32-48.

- `lists_all_transform(data)` (@noRd): Transform hook for ct_lists_all. Logic:
  1. Determine projection from data$params: if return_dtxsid is TRUE use "chemicallistwithdtxsids", else "chemicallistall"
  2. Call ct_chemical_list_all(projection = projection) to get the actual data
  3. Show cli::cli_alert_success("{nrow(df)} lists found!")
  4. If return_dtxsid AND coerce: split by listName, coerce dtxsids to vector (existing logic from ct_lists_all.R)
  5. If NOT return_dtxsid AND coerce: warn via cli
  6. Return the final df (or list if coerced)

  IMPORTANT: This is a transform hook — it REPLACES the default generic_request call. The generated stub for ct_chemical_list_all is a static endpoint. The transform hook wraps it with the projection/coerce logic.

- `format_compound_list_result(data)` (@noRd): Post-response hook for ct_compound_in_list. Extract the compound-in-list boolean/info from data$result, format with cli messages. Return the formatted result. Read the existing ct_compound_in_list.R to extract exact formatting logic before implementing.

**R/hooks/bioactivity_hooks.R:**

If FALSE, return data$result as-is. Extracted directly from ct_bioactivity.R annotate logic.

All hooks must follow the contract: receive list(params=..., result=...), return transformed data. Pre-request hooks don't have $result yet. Transform hooks return the final user-facing object.
