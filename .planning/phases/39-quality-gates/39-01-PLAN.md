<threat_model>
## Trust Boundaries

This phase crosses the boundary between local test execution and external ontology provider adapters. Tests must prove that package CI exercises provider parsing and degradation behavior without contacting OLS4, NVS, BioPortal, Wikidata, AGROVOC, or any other live provider. BioPortal key handling is also a credential boundary because CI should not require or leak a real API key.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39-01 | Information Disclosure | BioPortal credential handling | mitigate | Test missing-key behavior and use fake keys only; sentinel mocks fail if keyed tests try a live request |
| T-39-02 | Denial of Service | CI/provider tests | mitigate | Mock `httr2::req_perform()` and provider helpers so tests never depend on external provider availability |
| T-39-03 | Integrity | Provider candidate schema | mitigate | Assert the exact shared candidate schema on happy, failure, and empty paths |
| T-39-04 | Repudiation | Release validation evidence | mitigate | Use focused testthat command as the durable quality gate and avoid informal dev scripts |
</threat_model>
