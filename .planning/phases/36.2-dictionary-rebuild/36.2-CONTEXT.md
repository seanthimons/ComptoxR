**Gathered:** 2026-04-24

<domain>
## Phase Boundary

Full 139-term re-resolution of `lifestage_baseline.csv` through an expanded provider pipeline (alias normalization + OLS4-broader + BioPortal + NVS + Wikidata + AGROVOC), with auto-derived `lifestage_derivation.csv` rows using ontology-driven category mapping. <decisions>
## Implementation Decisions

### Adapter scope
- **D-01:** All 4 probed sources (OLS4-broader, BioPortal, Wikidata, AGROVOC) become permanent adapters wired into `.eco_lifestage_resolve_term()`. No source excluded based on probe hit rates — commit upfront.
- **D-02:** Permanent query order: Alias → OLS4(UBERON) → OLS4(PO) → BioPortal → NVS → Wikidata → AGROVOC. Broadest ontology sources first, specialist sources last.

### Alias/normalization layer
- **D-03:** Add an alias normalization step inside `.eco_lifestage_resolve_term()` that runs before any provider lookup. Data-driven from a committed CSV lookup table — not regex.
- **D-04:** Alias file committed as `inst/extdata/ecotox/lifestage_aliases.csv` alongside baseline/derivation/audit. Same `system.file()` access pattern. Installed with the package, editable by end users who run patch/rebuild.
- **D-05:** Alias file must be accessible in non-dev environments — any user rebuilding or patching the database can edit the file and use it to influence resolution.

### Resolution strategy
- **D-06:** Full 139-term re-resolution from scratch. All existing resolved rows are re-resolved through the expanded pipeline. Clean slate — existing resolutions may improve with broader search and alias normalization.
- **D-07:** 100% coverage gate: every one of the 139 terms must either resolve to an ontology ID or have a derivation row with an explicit, documented reason for remaining unresolved. No term left in limbo.
- **D-08:** Terms that cannot resolve to ontology IDs (out_of_scope_biology, ambiguous, etc.) still require a derivation row mapping them to an appropriate harmonized category (e.g., F0-F7 generations → Other/Unknown, microbial growth phases → Other/Unknown).

### Derivation authoring
- **D-09:** Auto-derive all — generate derivation rows for all resolved terms automatically using best-match logic from the resolution pipeline. This supersedes Phase 36's per-row curator-only model for this batch rebuild.
- **D-10:** Curator audits the final `lifestage_derivation.csv` as a whole before committing. One review pass over the complete file, not per-term approval.
- **D-11:** Harmonized category assignment uses ontology-driven mapping: resolved term's parent/ancestor in UBERON/PO hierarchy determines category (e.g., UBERON:post-embryonic → Juvenile). Build a lookup from ontology IDs to the 7 harmonized categories (Egg/Embryo, Larva, Juvenile, Subadult, Adult, Senescent/Dormant, Other/Unknown).

### Pipeline wiring
- **D-12:** All 4 new adapters are permanently wired into `.eco_lifestage_resolve_term()`. Future ECOTOX releases auto-query all sources via the permanent pipeline.
- **D-13:** Each adapter follows the existing tryCatch + `cli_warn` + empty tibble resilience pattern established in Phase 35. Individual provider failure does not kill the pipeline.
- **D-14:** Rate limiting per adapter: carry forward Sys.sleep patterns from probe scripts into permanent adapter code.

### Claude's Discretion
- Exact alias CSV columns beyond `org_lifestage` and `normalized_query`
- Ontology-to-category mapping table structure and location
- Adapter function naming (`.eco_lifestage_query_bioportal()`, `.eco_lifestage_query_wikidata()`, `.eco_lifestage_query_agrovoc()`)
- Whether OLS4-broader replaces or supplements the existing `.eco_lifestage_query_ols4()` function
- Validation script structure for Phase 36.2

### Policy artifact boundary
- **D-15:** Hand-authored lifestage policy must live in explicit package data artifacts, not hidden R literals. R code may implement loading, validation, scoring, and routing mechanics, but curated terms, forced-unresolved terms, domain patterns, and route-family mappings must be CSV-backed and tested.

</decisions>

<specifics>
## Specific Ideas

- The alias CSV should contain obvious synonym mappings informed by the audit CSV `notes` column (e.g., "Fish juvenile stage (pre-feeding salmon/trout after hatching)" for Alevin → query "alevin salmon juvenile hatching").
- Ontology-to-category mapping should handle the 7 harmonized categories defined in LIFESTAGE_HARMONIZATION_PLAN2.md: Egg/Embryo, Larva, Juvenile, Subadult, Adult, Senescent/Dormant, Other/Unknown.
- For out_of_scope_biology terms (F0-F7 generations, microbial growth phases), map to Other/Unknown with `derivation_source` explaining why.
- For ambiguous terms (Sexually mature, Young of year, "New, newly or recent hatch"), resolution should attempt ontology lookup first; if still ambiguous, curator assigns category during the audit pass.
- The `refresh_baseline.R` dev script should be updated to use the new expanded pipeline, so future ECOTOX releases get the same broad resolution coverage.

</specifics>
